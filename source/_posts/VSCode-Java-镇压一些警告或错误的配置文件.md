最近，发现之前用的 org.eclipse.jdt.core.prefs 配置并不完整，这是在打开《算法第四版》的配套代码的时候发现的，而且，原来我也没有去仔细找一下文档，去研究具体的每一个键值对的含义，这次，为了找到完整的配置文件和文档，着实花了不小的时间。这是当然不能不记录的。

那么，直截了当地，给出文档的地址，

<https://help.eclipse.org/latest/index.jsp?topic=%2Forg.eclipse.jdt.doc.isv%2Freference%2Fapi%2Forg%2Feclipse%2Fjdt%2Fcore%2FJavaCore.html>

这里的键值对配置项应该是比较全面的。如果我们需要暂时禁用某些警告，那么，在这里配置，把相应的 warning 改成 ignore 就可以了。至于 error 值，我们当然是不可以 suppress 的。

本来其实我也有看到利用 maven 的正则替换的插件来给每一个 class 类加上 `@SuppressWarnings("all")` 的，但是，这样，其实是很不优雅的。只有实在没有办法的时候才可以用这个方法。

可以参考这个 [issue](https://github.com/redhat-developer/vscode-java/issues/1146)，把相应的键设置成空值就可以了，

```shell
org.eclipse.jdt.core.compiler.taskTags=
```
